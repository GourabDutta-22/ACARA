Formal Definitions
Step 1
Query Encoding
eq = φ(q) ∈ ℝd
φ : OpenAI text-embedding-ada-002 · d = 1536
Step 2
Adaptive Retrieval
R = TopK(V, eq, karc)
V = vector store (Pinecone / ChromaDB)
karc ∈ [2, 8], default = 3
Step 3a
Similarity Gate
S(q, di) = sim(eq, edi) = 1 − distcos ≥ τarc
τarc ∈ [0.35, 0.60], default = 0.45
Pass if ∃ di ∈ R such that S ≥ τ
Step 3b
Freshness Gate
F(di) = 𝟙[tnow − tstored(di) ≤ 72h]
Stale if > 50% of chunks fail: |{di : ¬F(di)}| > |R| / 2
Step 3c
LLM Coverage Gate
C(di, q) = LLMjudge(di, q) ∈ {yes, no}
Evaluated on top-2 documents only
Pass if ∃ di : C(di, q) = yes
Step 3
Context Awareness Gate — Combined Decision
G(q, R) = S ∧ C ∧ F
G = 1 (Strong) → route to Credibility
G = 0 (Weak)  → route to WebSearch → Credibility
Feedback
ARC Feedback Update Rules
When G = 0 (Weak Signal)
τ(t+1) = max(0.35, τ(t) − 0.05)
k(t+1) = min(8, k(t) + 1)
When G = 1 (Strong Signal)
τ(t+1) = min(0.60, τ(t) + 0.02)
k(t+1) = max(3, k(t) − 1)
Dynamic chunk sizing: |q| > 20 words → s ← max(300, s − 500),   |q| < 6 words → s ← min(2000, s + 500)
Step 4
Credibility Scoring
Pcred(di | q) = LLMscore(di, q)
Dcredible = {di ∈ D : Pcred = yes}
Only credible chunks are written to long-term Vector Memory
Step 5–6
Generation & Critic Validation
r = LLM(ctx, q, H)
r* = r · 𝟙[Critic(r, ctx) = pass]
ctx = ranked context from Dcredible
H = conversation history (MongoDB)
Critic checks: hallucination, grounding, relevance

Appendix A
System Workflow Pseudocode
ARC Adaptive Retrieval Controller — Feedback-Driven Parameter Adjustment
class ARCController: // Default parameters (initialized at session start) similarity_threshold ← 0.45 top_k ← 3 chunk_size ← 1000 chunk_overlap ← 200 function adjust_on_weak_context(): // Called when Context Awareness Gate signals WEAK similarity_threshold ← max(0.35, threshold − 0.05) // broaden recall top_k ← min(8, top_k + 1) // retrieve more chunks function adjust_on_strong_context(): // Called when Context Awareness Gate signals STRONG similarity_threshold ← min(0.60, threshold + 0.02) // tighten precision if top_k > 3: top_k ← top_k − 1 // decay toward default function adjust_chunk_size(query): word_count ← len(query.split()) if word_count > 20: // complex query chunk_size ← max(300, chunk_size − 500) // smaller, focused chunks overlap ← max(50, overlap − 25) elif word_count < 6: // simple query chunk_size ← min(2000, chunk_size + 500) // larger context windows overlap ← min(400, overlap + 25)
ROUTING Pipeline Routing Decisions (Context Awareness Gate → Conditional Edges)
function grade_documents_node(state): documents ← state.documents distances ← state.distances threshold ← ARC.similarity_threshold // Gate 1: Similarity Check similarity_pass ← any( (1 − d) ≥ threshold for d in distances ) // Gate 2: Freshness Check (72-hour staleness window) freshness_ok ← stale_count ≤ len(metadatas) / 2 // Gate 3: LLM Coverage Verification (top-2 documents) coverage_pass ← false if similarity_pass: for doc in documents[:2]: if LLM.judge_coverage(doc, query) == "yes": coverage_pass ← true break context_strong ← similarity_pass AND coverage_pass AND freshness_ok if context_strong: ARC.adjust_on_strong_context() route → credibility_node // proceed to Credibility Scoring else: ARC.adjust_on_weak_context() route → web_search_node // fallback to External Knowledge // Full Pipeline Graph (LangGraph StateGraph): // retrieve → grade → [web_search | credibility] → generate → critic → END // ↳ web_search → credibility → generate → critic → END

Hyperparameter Recurrence Relations
θ₁
Similarity Threshold — τarc
Weak Context (G = 0)
τ(t+1) = max(τmin, τ(t) − Δτ↓)
= max(0.35, τ(t) − 0.05)
Strong Context (G = 1)
τ(t+1) = min(τmax, τ(t) + Δτ↑)
= min(0.60, τ(t) + 0.02)
Domain: τ ∈ [0.35, 0.60]   ·   Initialization: τ(0) = 0.45   ·   Step ratio: Δτ↓ / Δτ↑ = 0.05 / 0.02 = 2.5× (aggressive broadening, conservative tightening)
Usage: cosine similarity check — S(q, di) = 1 − distcos(eq, edi) ≥ τarc
Convergence: After n consecutive weak signals, τ(n) = max(0.35, 0.45 − 0.05n).   Saturates at floor after 2 weak signals. After n consecutive strong signals, τ(n) = min(0.60, 0.45 + 0.02n).   Saturates at ceiling after 8 strong signals.
θ₂
Top-K Retrieval Count — karc
Weak Context (G = 0)
k(t+1) = min(kmax, k(t) + 1)
= min(8, k(t) + 1)
Strong Context (G = 1)
k(t+1) = max(kdefault, k(t) − 1)
= max(3, k(t) − 1)
Domain: k ∈ {2, 3, 4, 5, 6, 7, 8} ⊂ ℤ   ·   Initialization: k(0) = 3   ·   Decay floor: kdefault = 3 (not kmin = 2)
Note: Strong signal decays k toward 3 (default), not toward 2 (minimum). This preserves baseline retrieval depth.
Retrieval set: R = {d₁, d₂, ..., dk} where di are ranked by cosine similarity to eq.   More chunks (↑k) increases recall but adds noise; fewer (↓k) improves precision.
θ₃
Dynamic Chunk Size — sarc and Overlap — oarc
Complex Query (|q| > 20 words)
s(t+1) = max(smin, s(t) − Δs)
= max(300, s(t) − 500)
o(t+1) = max(50, o(t) − 25)
Simple Query (|q| < 6 words)
s(t+1) = min(smax, s(t) + Δs)
= min(2000, s(t) + 500)
o(t+1) = min(400, o(t) + 25)
Domain: s ∈ [300, 2000] chars   ·   Overlap: o ∈ [50, 400] chars   ·   Initialization: s(0) = 1000, o(0) = 200
Trigger condition: |q| = word count of query string;   6 ≤ |q| ≤ 20 → no adjustment
Chunking function: RecursiveCharacterTextSplitter(s, o, separators=["\n\n", "\n", ". ", "! ", "? ", " "])
Intuition: Complex multi-part queries benefit from smaller, focused chunks for precise retrieval. Simple keyword queries benefit from larger context windows to capture broader information. The overlap parameter maintains continuity between adjacent chunks to prevent information loss at boundaries.
Σ
Combined ARC State Transition Function
ARC(t+1) = Ψ(τ(t), k(t), s(t), o(t), G(t), |q(t)|)
where   Ψ : [0.35, 0.60] × {2…8} × [300, 2000] × [50, 400] × {0, 1} × ℤ+ → ARC(t+1)
All parameters are bounded (clamped) and self-correcting.   The system converges to stable operating points under consistent feedback signals.   Reset: ARC(0) = (0.45, 3, 1000, 200) at each new session.

Validated Response
The Adaptive Retrieval Controller (ARC) serves as the self-optimizing core of ACARA. It dynamically tunes four key retrieval parameters — similarity threshold, top-k, chunk size, and overlap — based on real-time feedback from the Context Awareness Gate. When retrieved context is judged weak, ARC broadens the search by lowering the similarity threshold (−0.05) and retrieving more chunks (+1). Conversely, when context is strong, it tightens precision (+0.02) and decays top-k back toward the default. This closed-loop mechanism ensures retrieval quality improves with each successive query.

Validated Response
ACARA handles stale documents through its 3-gate Context Awareness system. The Freshness Check gate examines the ISO timestamp of each retrieved chunk against a 72-hour staleness window (FRESHNESS_HOURS = 72). If more than half the retrieved chunks exceed this age threshold, the freshness flag is set to false, and the overall context is judged as WEAK — triggering the web search fallback to obtain fresh information. Additionally, newly retrieved web content is scored for credibility before being stored in Vector Memory, ensuring the knowledge base is continuously refreshed with verified, up-to-date information.
